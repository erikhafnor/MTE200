Ultrasound Laboratory – GE LOGIQ S8 & IHE Scheduled Workflow

Learning Objectives

  • Perform and optimize an ultrasound examination: image optimization, quantitative measurements, and structured analysis.
  • Execute an end‑to‑end imaging workflow using IHE Scheduled Workflow components (HIS → RIS/PACS → Modality → Workstation) including HL7 ADT and DICOM MWL & Storage.
  • Conduct performance / quality tests on an ultrasound system using a Kyoto Kagaku N‑365 phantom (axial, lateral/angular resolution, near-field resolution, cyst sizing, string target spacing).
  • Document and archive images into a PACS and retrieve them via a diagnostic viewer (OHIF) for verification and longitudinal QA.
  • Carry out a limited functional test on a volunteer (vascular scanning – carotid or radial artery) ethically and safely.
  • Understand and explain key imaging parameter controls (Dynamic Range, CrossBeam, Focus Width, Suppression, Line Density) and probe label nomenclature.

Safety & Handling

  • Handle probes carefully – acoustic lens is delicate; always use ultrasound gel (never scan the phantom dry).
  • Secure probe cables in designated holders to avoid strain and tripping hazards.
  • Use only approved coupling gel with the phantom; wipe excess gently after use.
  • Volunteer scanning: obtain verbal consent; no diagnostic intent, minimal exposure, follow infection control (probe covers if scanning vascular access sites).

IHE Scheduled Workflow Overview

The Scheduled Workflow (SWF) profile coordinates patient demographics & orders (HL7 ADT/ORM) with Modality Worklist (MWL) query and subsequent DICOM image storage and query/retrieve. Sequence:

  1. Create patient (ADT) in HIS (HAPI Test Panel).
  2. Send HL7 message to RIS/PACS (DCM4CHE) – creates order/scheduling record.
  3. On the ultrasound (Modality) perform DICOM MWL query; select the scheduled patient/work item.
  4. Acquire phantom and (later) volunteer images; store to PACS (DICOM C‑STORE).
  5. Query & retrieve in OHIF viewer (DICOM Q/R) to confirm archival integrity.

Part A – Worklist & Patient Setup

  1. Open HAPI Test Panel; load provided HL7 ADT file (Canvas) with given demographics; transmit to RIS/PACS.
  2. On LOGIQ S8, perform MWL query; verify patient appears; select and begin exam from worklist (ensures demographic consistency in DICOM headers).

Part B – Phantom Performance Tests (Kyoto Kagaku N‑365)

For each test record settings: transducer type, frequency (fundamental / harmonic), depth, focus position(s), dynamic range, gain (overall / TGC), line density / CrossBeam status, image preset.

  1. Axial Resolution: Increase frequency; optimize focus at target depth; measure minimum separable vertical (depth) spacing of two reflectors.
  2. Angular (Lateral) Resolution: Use appropriate phantom targets; vary focus and beam compounding (CrossBeam) to observe lateral separation capability.
  3. Near-Field Resolution: Evaluate shallow targets; adjust frequency and focus; note degradation due to ring-down / dead zone; record minimum detectable structure depth.
  4. Cyst (Anechoic) Measurement: Measure depth & diameter (calipers) for several cysts; compare to phantom specification; compute % error.
  5. String Targets Spacing: Measure inter-string distances; compare with manual spec; record deviation.
  6. Capture and store representative images for each test (include annotations / measurement calipers).

Part E – Limited Functional Test (Volunteer Vascular Scan)

  1. Start a new exam (not tied to phantom order—note patient volunteer identifier anonymized).
  2. Select linear probe; optimize for superficial vascular imaging (appropriate frequency & depth).
  3. Scan carotid (or radial) artery in longitudinal & transverse planes; adjust gain, dynamic range, focus to clearly delineate vessel wall.
  4. Optional: Activate Doppler (color / spectral) to illustrate flow; note angle correction (≤60°) and sample volume placement.
  5. Capture representative images (B‑mode and Doppler) – store but ensure non-clinical labeling.

Key Parameter Definitions

  • Dynamic Range: Logarithmic compression range mapping echo amplitude to grayscale; lower values increase contrast but risk saturation.
  • CrossBeam (Spatial Compounding): Averages multiple steering angles reducing speckle and edge shadowing; may decrease frame rate and reduce appearance of certain specular reflections.
  • Focus Width: Lateral aperture size / number of elements used at focus; wider can improve lateral resolution at focus but reduce frame rate and increase sidelobes.
  • Suppression (Speckle / Noise Reduction): Adaptive filtering reducing high-frequency noise; excessive use can blur fine texture or small lesion margins.
  • Line Density: Number of scan lines per frame; higher improves lateral resolution & contrast resolution but lowers frame rate.

Probe Label Nomenclature (Examples)

  • Lxx: Linear array, xx MHz center or frequency range emphasis (e.g., L12‑5).
  • Cxx: Curvilinear array for abdominal/OB (e.g., C5‑2).
  • P or S / Sector: Phased array for cardiac intercostal windows (e.g., P4‑2).

Analysis & Reflection Questions

  1. How did adjusting dynamic range affect cyst detection and string target contrast? Provide before/after image references.
  2. Quantify improvement (if any) in lateral resolution (Δx) with increased line density and CrossBeam enabled versus baseline.
  3. Discuss trade-offs between frame rate and spatial resolution for vascular Doppler imaging.
  4. Explain why axial resolution primarily depends on pulse length (bandwidth) while lateral depends on aperture & focusing.
  5. Identify potential systematic errors in phantom measurements (speed of sound assumption, caliper placement, probe pressure) and estimate magnitude.
  6. Evaluate whether stored DICOM metadata correctly preserved measurement annotations; note any missing structured reporting elements.
  7. For the volunteer scan, comment on image optimization strategies ensuring clear intima-media delineation.

Common Pitfalls

  • Forgetting to perform MWL query leading to manual patient entry mismatch.
  • Not saving images immediately after optimization (losing settings context).
  • Excess probe pressure altering phantom geometry or vascular lumen shape.
  • Overusing suppression leading to loss of low-contrast lesion visibility.
  • Failing to note line density and CrossBeam status in measurement tables (difficult to reproduce).
